## 🎓 经验总结

### 主要教训

1. **数据库一致性至关重要** - 表结构不一致导致多个接口 500 错误
2. **认证必须默认开启** - 初始代码所有 CRUD 无认证是严重安全隐患
3. **共享资源管理** - 每请求新建数据库连接会导致资源耗尽
4. **SQL 引号规范** - SQLite 双引号用于标识符，字符串用单引号
5. **环境变量管理** - 密钥不应硬编码在代码中

### 最佳实践

1. ✅ 使用中间件统一管理认证逻辑
2. ✅ 数据库连接复用，避免泄漏
3. ✅ 参数化查询防止 SQL 注入
4. ✅ 输入验证在前端和后端都进行
5. ✅ 公开接口和受保护接口明确区分
6. ✅ 错误信息不暴露敏感数据
7. ✅ 密码使用 bcrypt 加密存储
8. ✅ 文档与代码同步更新
